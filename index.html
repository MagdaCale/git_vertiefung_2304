Git Vertiefung

  1. Erstelle ein Projekt & eine Repo
  2. Lade das Team ein
  3. Wir erstellun uns eine Branch
    • git branch feature -> legt eine neue branch an
    • git checkout feature -> wechselt zur branch feature
    • git branch -b feature2 -> legt eine neue Branch an und wechselt direkt dorthin
    • git branch -> zeigt alle vorhanden branches an (* -> zeigt an, wo man sich gerade befindet)
    • git branch -d feature2 -> löscht den branch feature2
    • git branch -D feature2 -> Branch wird unabhängig von seinem Status und ohne Warnung gelöscht

Commits

  • git log --all -> zeigt alle commits an
  • git log --oneline -> zeigt alle commits in einer Zeile
  • git log --graph -> zeigt dir alle commits grafisch an
  • mit "q" kommst du aus der Ansicht wieder raus

Das Mergen

Sobald sämtliche Teams ihre Änderungen erfolgreich in den main/dev branch eingebracht haben und dieser auf dem neuesten Stand ist, ist nun an der Zeit, die Inhalte des main/dev branches in unseren eigenn Zweig zu mergen

  1. Falls du dich nicht in deiner branch befindest -> git checkout "branch name"
  2. git merge main -> main wird mit deiner branch gemerged
  3. lösche deine branch, wenn du sie nicht mehr benötigst -> git branch -d "branch name"
